# Mistral AI / TanStack Bulk npm and PyPI Supply Chain Attack — JavaScript/Python

## 📅 Incident Date
May 11, 2026 (5-hour attack window)

## 🎯 Target System
**npm and PyPI packages**. 170+ npm packages and 2 PyPI packages were targeted, totaling 404 malicious versions.
- @tanstack/react-router (3M+ weekly npm downloads)
- @mistralai/mistralai (official Mistral AI SDK)
- @opensearch-project/opensearch (1.3M weekly downloads)
- @uipath/robot (UiPath RPA automation)
- @tanstack/vue-router, @tanstack/solid-router
- mistralai (PyPI) and guardrails-ai (PyPI)

## 🔴 What Happened?
A large-scale, coordinated supply chain attack targeting multiple ecosystems:

**How it worked:**
1. The attacker targeted the entire npm scope of 170+ packages
2. Published 1-18 malicious versions per package (404 total versions)
3. In Mistral AI packages: `preinstall` hook was used — legitimate build scripts were removed, replaced with `node setup.mjs`
4. In TanStack packages: `optionalDependency` mechanism was used — pointing to a malicious commit in the real tanstack/router repo
5. The payload downloaded and ran the Bun runtime

**What did the payload do?**
- Collect AWS IAM credentials
- Collect HashiCorp Vault tokens
- Collect GitHub tokens (`ghp_`, `gho_`, `ghs_`)
- Collect npm publish tokens
- Collect GitHub Actions OIDC tokens
- Commit poisoned `.claude/settings.json` and `.vscode/tasks.json` files via stolen tokens through GitHub GraphQL API
- Data was exfiltrated via the Session (onion-routed) protocol

**PyPI variant:**
- `mistralai==2.4.6` and `guardrails-ai==0.10.1`
- A different payload: downloaded and ran `git-tanstack[.]com/transformers.pyz` during Python import
- Same `git-tanstack[.]com` domain was used

## 🧠 Root Cause
1. **Weak npm account security** — attackers compromised maintainer accounts (exact method unknown)
2. **Inadequate npm publish rights protection** — all packages in a scope could be published simultaneously
3. **preinstall hooks** — npm allows arbitrary script execution during installation
4. **Similar PyPI security vulnerabilities** — code execution during import
5. **Insufficient package validation** — packages not security-audited before publication

## 💥 Impact
- **170+ npm packages**, **2 PyPI packages** compromised
- **TanStack Router ecosystem**: 42 packages, 84 versions fully compromised
- **Mistral AI**: 3 SDK packages, 9 versions
- **UiPath**: 65 packages, 65 versions
- **OpenSearch**: 1.3M weekly downloads, 4 versions
- Guardrails AI and Mistral AI PyPI projects quarantined
- Cloudflare flagged `git-tanstack[.]com` domain as phishing
- Self-spreading via poisoned GitHub repos using stolen credentials

## 🔧 How Was It Resolved?
- SafeDep, StepSecurity, and Socket Security detected and reported the attack (named "mini-shai-hulud")
- PyPI projects (mistralai, guardrails-ai) quarantined
- npm removed the malicious packages
- Cloudflare flagged the domain as phishing
- Emergency warnings issued to TanStack and Mistral AI users
- Tools like Package Manager Guard (PMG) recommended for npm

## 🎓 Lessons Learned
1. **A single attack can target the entire ecosystem** — multiple package managers (npm, PyPI) simultaneously
2. **preinstall/postinstall scripts carry significant risk** — this feature should be questioned from a security perspective
3. **Package age and popularity are not guarantees of safety** — well-known, popular packages can still be compromised
4. **Delayed installation (cooldown policy)** — don't install newly published packages immediately, wait
5. **AI-based threat detection** — tools like SafeDep can detect abnormal publication patterns

## Vibe Coding Connection: How this mistake can be repeated in AI code generation
This attack becomes much more dangerous in the AI era. AI code assistants (Claude Code, Cursor, GitHub Copilot) can automatically run `npm install` or `pip install` on behalf of a developer. An AI tool could install dependencies without asking the developer. Even worse, packages popular in AI training data might be targeted. At the same time, `.vscode/tasks.json` or `.claude/settings.json` files generated by AI could be poisoned, as in this attack. Limit what commands AI agents can execute.
