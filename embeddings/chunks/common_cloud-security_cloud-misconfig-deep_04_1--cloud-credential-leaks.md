## 1. Cloud Credential Leaks in AI-Generated Code

AI code assistants (GitHub Copilot, Cursor, Claude Code, etc.) frequently generate code with hardcoded credentials. Research shows this is a growing epidemic:

- **28-29 million** secrets leaked on GitHub in 2025 (Snyk, 2026)
- **45%** of AI-generated code samples introduce OWASP Top 10 vulnerabilities (Cloud Security Alliance, 2026)
- AI assistants often **generate dummy credentials that are real or guessable** (e.g., `password123`, `sk-test-...`)

### AI-Generated Vulnerable Code (Claude Code / Copilot)

```python
# AI-GENERATED VULNERABLE CODE: Hardcoded AWS credentials
# Often generated by AI assistants as "quick examples"

import boto3

# AI assistant generated this:
AWS_ACCESS_KEY_ID = "AKIAIOSFODNN7EXAMPLE"  # Real AWS key pattern!
AWS_SECRET_ACCESS_KEY = "wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY"  # Leaked!

s3 = boto3.client(
    's3',
    aws_access_key_id=AWS_ACCESS_KEY_ID,
    aws_secret_access_key=AWS_SECRET_ACCESS_KEY
)

# The user commits this, and automated scanners find the keys
```

### Why AI Generates Credential Leaks

| Cause | Explanation |
|---|---|
| **Training data leakage** | AI models trained on public code including leaked credentials |
| **Lack of security awareness** | AI optimizes for "it works" not "it's secure" |
| **No context of production** | AI doesn't know this is for a real deployment |
| **Hardcoded defaults** | AI replicates bad patterns from documentation |

### Secure Alternative (AI-Suggested with Env Vars)

```python
# SECURE: Always use environment variables for credentials
import os
import boto3
from botocore.exceptions import NoCredentialsError

# Get credentials from environment or IAM role
# NEVER hardcode — use AWS IAM roles in production
def get_s3_client():
    """Initialize S3 client using IAM role (EC2/Lambda) or env vars."""
    try:
        # Prefer IAM role (works in EC2, Lambda, ECS)
        session = boto3.Session()
        s3 = session.client('s3')
        # Test connection
        s3.list_buckets()
        return s3
    except NoCredentialsError:
        # Fall back to environment variables (local dev only)
        access_key = os.environ.get('AWS_ACCESS_KEY_ID')
        secret_key = os.environ.get('AWS_SECRET_ACCESS_KEY')
        region = os.environ.get('AWS_REGION', 'us-east-1')
        
        if not access_key or not secret_key:
            raise ValueError(
                "AWS credentials not found. Use IAM roles on cloud, "
                "or set AWS_ACCESS_KEY_ID/AWS_SECRET_ACCESS_KEY env vars locally."
            )
        
        return boto3.client(
            's3',
            aws_access_key_id=access_key,
            aws_secret_access_key=secret_key,
            region_name=region
        )
```
