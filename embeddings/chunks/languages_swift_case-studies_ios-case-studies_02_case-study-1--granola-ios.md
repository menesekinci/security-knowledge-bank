## Case Study 1: Granola iOS API Key Exposure (TRA-2025-07)

### Incident Overview
- **Date**: December 2024 – March 2025
- **Platform**: iOS (TestFlight beta)
- **Researcher**: Ben Smith (Tenable)
- **Advisory**: TRA-2025-07
- **Severity**: Critical (hardcoded API key could leak private meeting transcripts)

### Description
Granola, an AI-powered note-taking app, exposed an AssemblyAI API key in their iOS TestFlight beta app. The key was hardcoded in the app's API endpoint response used for feature flags and configuration. If exploited, an attacker could have accessed meeting transcripts generated by AssemblyAI for 333 beta testers.

### Root Cause
1. An AssemblyAI API key was exposed through the app's feature-flag/configuration API endpoint
2. The key was accessible to anyone who inspected the app's network traffic
3. Even after transcription moved to the backend (Dec 16, 2024), the key remained unintentionally accessible

### Timeline
| Date | Event |
|---|---|
| Nov 14, 2024 | Granola iOS beta begins (invite-only TestFlight) |
| Nov 27, 2024 | AssemblyAI key exposed via API endpoint |
| Dec 16, 2024 | Transcription moved to backend (key remained accessible) |
| Mar 11, 2025 | Tenable disclosed vulnerability → Granola revoked key same day |

### Impact
- 333 beta testers' meeting transcripts potentially exposed
- No evidence of data theft; Tenable's controlled test was the only access
- Production macOS app was unaffected (uses different transcription service)

### Lessons for iOS Developers
1. **Never embed API keys in client-side apps** — anything in the app binary or API responses is accessible
2. **Use Apple's App Attest or device checks** to restrict API access to genuine app instances
3. **Audit API endpoints** for leaked secrets in responses
4. **Implement proper secret management**: proxy third-party API calls through your backend

### Vulnerable Pattern
```swift
// VULNERABLE: API key exposed via configuration endpoint
// /api/config returns:
{
  "assemblyai_key": "abc123def456",  // API key in plaintext!
  "feature_flags": { "transcription": true }
}
```

### Secure Pattern
```swift
// SECURE: Server-side proxy with device attestation
// /api/transcribe (client sends audio, server adds API key)
func transcribeAudio(audioData: Data) async throws -> String {
    var request = URLRequest(url: URL(string: "\(API_BASE)/transcribe")!)
    request.httpMethod = "POST"
    request.httpBody = audioData
    request.setValue("application/octet-stream", forHTTPHeaderField: "Content-Type")
    
    // Attest device identity
    let attestation = try await generateAttestation()
    request.setValue(attestation, forHTTPHeaderField: "X-Device-Attestation")
    
    let (data, _) = try await URLSession.shared.data(for: request)
    return try JSONDecoder().decode(TranscriptionResponse.self, from: data).text
}
```
